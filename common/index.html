Common

This module defines some important "helper" types used by the solver.

SetGuardedByLits 
SetGuardedByLits()

Bases: Generic[T]

Represents a "guarded" collection of elements. Each element is guarded by a literal (and implicitly all literals stronger than it).

Implemented as a simple wrapper around a Dict[SignedVar, Dict[Bound, Set[T]]].

Note

This class is most notably used for the implication graph on non optional variables in Solver, as well as for (watched) literals watches in SATReasoner and DiffReasoner.

add 
add(element: T, literal: Lit) -> None

Adds a new guarded element to the collection.

Raises:

  • ValueError –

    If element is already present and already guarded by literal (or, implicitly, any literal stronger than it).

Note

An element can be added if it is absent from the collection, or if it is guarded by a literal stronger than literal. When we record this new, weaker guard, the previous guard becomes redundant. However we make no effort to erase it, as it does not harm or interfere with anything functionality of the class.

remove 
remove(element: T, literal: Lit) -> None

Removes a guarded element from the colleciton.

Raises:

  • ValueError | KeyError –

    If element is not in the collection or is not guarded by literal.

remove_all_on 
remove_all_on(signed_var: SignedVar) -> None

Removes all elements guarded by any literal on the given signed variable.

Raises:

  • KeyError –

    If no guard literal on signed_var is registered.

Returns:

  • Tuple[T, ...] –

    All elements guarded by literal (and, implicitly, all literals stronger than it). The elements returned are unique, there are no duplicates.

Encoding

Bases: NamedTuple

REVIEW? Represents a cause corresponding to the deactivation of a scope in which an (already reified) constraint, found to be impossible to satisfy, is defined. In other words, this means setting the corresponding scope literal as false.

ImplicationPropagation

Bases: NamedTuple

Represents a cause corresponding to an implication propagation.

Note

Implication propagations are triggered on bound updates of non optional variables (notably presence variables) and concern literals that are implied by the newly entailed literal. This is possible thanks to the implication graph on non optional variables' literals maintained by the solver.

literal instance-attribute 
literal: Lit

The literal whose entailment triggered the implication propagation.

EmptyDomain

Bases: NamedTuple

Represents a cause corresponding to the prevention of an optional variable's domain becoming empty, by setting its presence literal to false.

Note

Presence variables are non optional by definition.

literal instance-attribute 
literal: Lit

The literal whose entailment would make its variable's domain empty.

cause instance-attribute 
cause: Causes.AnyCause

The cause of the attempt to entail the literal that would make its variable's domain empty.

Event

Bases: NamedTuple

Represents an event, i.e. (meta)data on the entailment of a literal (or, equivalently, a signed variable's bound update).

signed_var instance-attribute 
signed_var: SignedVar

The signed variable whose bound was updated with the event.

bound instance-attribute 
bound: Bound

The new bound of the event's signed variable. It is necessarily strictly stronger than its previous bound.

old_bound instance-attribute 
old_bound: Bound

The previous bound that this event's signed variable had, before the event.

index instance-attribute 
index: Tuple[int, int]

The index of the event (see Solver.events_trail).

Note

It is a "double index": the first int of the tuple corresponds to the decision level, and the second one corresponds to the event's index in that decision level.

old_index instance-attribute 
old_index: Optional[Tuple[int, int]]

The index of the previous event on the same signed variable (if not None). This previous event set the signed variable's old bound. A None value indicates that there is no earlier events on the signed variable (i.e. the event is the first one on the signed variable).

cause instance-attribute 
cause: Causes.AnyCause

The cause of this event.

InvalidBoundUpdate

Bases: NamedTuple

Represents an encountered conflict, corresponding to an invalid bound update (i.e. an attempt to entail a literal which would cause its variable's domain to become empty).

It will be analyzed as part of conflict analysis, to produce a clause for the solver to "learn".

literal instance-attribute 
literal: Lit

The literal whose entailment would cause its variable's domain to become empty.

cause instance-attribute 
cause: Causes.AnyCause

The cause for the attempt to make the invalid bound update.

ReasonerBaseExplanation

Bases: NamedTuple

Represents a "starting point" or "base" explanation for a conflict encountered by a reasoner.

It will be refined into a full explanation as part of conflict analysis, to produce a clause for the solver to "learn".

literals instance-attribute 
literals: Tuple[Lit, ...]

The literals of the base / "starting point" explanation made by the reasoner.

ConflictAnalysisResult

Bases: NamedTuple

Represents the results of conflict analysis, namely the clause for the solver to "learn" (i.e. integrate in a SATReasoner after backtracking to an appropriate earlier decision level - in our case, the 1st Unique Implication Point (1UIP)) to prevent the analyzed conflict from being encountered again.

asserting_clause_literals instance-attribute 
asserting_clause_literals: Tuple[Lit, ...]

The literals composing the asserting clause obtained as a result of conflict analysis.

resolved_literals instance-attribute 
resolved_literals: Dict[SignedVar, Bound]

The resolved literals that participate in the conflict. Stored as a dictionary instead of a tuple of literals.

These literals appeared in an explanation when producing the asserting clause, but were "recursively" replaced by their own explanation (and thus do not appear in the clause).

They are typically exploited by some branching heuristics (e.g. LRB) to identify literals participating in conflicts.
